The core pixel-capture → dHash matching → popup pipeline is already platform-agnostic. The YouTube work is about:

2. Guarding Twitch-specific code paths (Phase 2 — next)

Key architectural insight: `findBestVideo()`, `clientToVideoCoords()`, and `captureRegion()` are all already platform-agnostic. The Twitch-specific code is limited to:
- `detectTwitchGame()` — DOM scraping for category link
- `detectTwitchExtensions()` / `disableTwitchExtensions()` — iframe overlay handling
- Popup tab.status check — Twitch URL detection

All of these can be platform-guarded with a simple `PLATFORM` constant derived from `location.hostname`.
