<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Extension popup → external URL | The "Privacy →" link navigates the user's browser to an external origin. |
| STORE-LISTING.md → CWS review | Text claims about privacy/data handling are public and auditable. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Spoofing | popup.html anchor tag | mitigate | Use `rel="noopener noreferrer"` on the anchor so the opened tab cannot navigate the popup window via `window.opener`. Already required in task action. |
| T-13-02 | Information Disclosure | STORE-LISTING.md privacy claims | mitigate | Claims must be accurate: verify outbound requests are limited to raw.githubusercontent.com, cdn.jsdelivr.net, and workers.dev (already declared in manifest host_permissions). No claim of "zero network" — only "no data leaves device during normal use." |
| T-13-03 | Tampering | External privacy policy URL | accept | `frothydv.github.io/streamGenie/privacy` is a GitHub Pages site controlled by the repo owner. No MITM risk beyond standard HTTPS trust model. Low risk for a documentation link. |
| T-13-04 | Elevation of Privilege | chrome.tabs.create absence | accept | Task 1 uses an anchor tag rather than chrome.tabs.create, so no additional API surface is introduced. Risk accepted — no new permissions needed. |
</threat_model>
