  <behavior>
    - hammingDistance(hashA: Uint8Array, hashB: Uint8Array) → number: count element-wise differences (each element is 0 or 1 since dHash output is binary bits packed as bytes)
    - makeTriggerRawId(name: string) → string: name.toLowerCase().replace(/[^a-z0-9]+/g, '-').replace(/^-|-$/g, '')
    - deduplicateTriggers(newTriggers, existingProfile, pngsByTriggerId): merges new triggers into existing profile
      - newTriggers: Array<{name, cropBuf: Buffer, cropW, cropH, nccScore, confidence, sourceTimestamp, frameW, frameH}>
      - existingProfile: the profile.json object (or null for first run)
      - pngsByTriggerId: Map<triggerId, Buffer> for looking up existing reference PNGs (to compute existing hash)
      - For each new trigger:
        1. Compute triggerId = makeTriggerRawId(name)
        2. Look up in existingProfile.triggers by id
        3. If NOT found → new trigger, add to result
        4. If found → compute dHash of new crop; compute dHash of existing reference[0]; hammingDistance ≤ 8 → skip (duplicate, log "Skipping duplicate: {name}"); > 8 → add cropBuf as additional reference on existing trigger (variant, log "Adding variant: {name}")
      - Returns { triggers: merged trigger array, newCount, variantCount, skipCount }
    - Trigger schema output matches CLAUDE.md profile.json schema: { id, payloads: [{title, text, popupOffset, image}], references: [{file, w, h, srcW, srcH}], rotation: {mode:'orthogonal'}, rotates: false }
    - popupOffset defaults to { x: 0, y: 0 } for AI-generated triggers (user can edit later)
    - reference.file = "{triggerId}-{refIndex}.png" (0-indexed)
    - Exports: { deduplicateTriggers, hammingDistance, makeTriggerRawId }
  </behavior>
  <action>
    Create scripts/ai-populate/lib/dedup.js. The dHash computation requires pngjs to decode the PNG buffer and matcher-core.js for the hash. Import both at top: const { PNG } = require('pngjs'); const MatcherCore = require('../../../extension/matcher-core.js'); const matcher = MatcherCore.createMatcher();

    For computing dHash of a PNG buffer: PNG.sync.read(buf) → pixels; matcher.dHashFromPixels(new Uint8Array(png.data.buffer, png.data.byteOffset, png.data.byteLength), png.width, png.height) → Uint8Array(64).

    The pngsByTriggerId map is populated by the orchestrator from the existing branch profile; dedup.js receives it as a parameter. If an existing trigger has no PNG in the map (e.g. first-run or PNG not passed), skip the hash check and treat as non-duplicate (conservative: better to add a variant than silently drop).

    Create tests/ai-populate/test-dedup.js. Tests:

    4. makeTriggerRawId('STS2: Map!') === 'sts2-map'

    6. deduplicateTriggers with same name and hammingDistance ≤ 8 → skipCount === 1, triggers unchanged
    7. deduplicateTriggers with same name and hammingDistance > 8 → variantCount === 1, trigger.references.length === 2
    Use synthetic solid-color PNG buffers to produce deterministic dHash values for tests 5–7 (same color → identical hash; different colors → different hash).
