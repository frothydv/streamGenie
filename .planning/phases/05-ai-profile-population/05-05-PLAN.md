  1. PREFLIGHT, 2. FRAME EXTRACTION, 3. WIKI GROUNDING,

  6. DEDUP, 7. PROFILE BRANCH BUILD, 8. VALIDATION PASS,
  9. PR CREATION, 10. SUMMARY REPORT

    Create scripts/ai-populate/lib/report.js. Export { generateReport, writeReport }.

    generateReport(stats) takes a stats object and returns a markdown string. Required stats fields:
    - youtubeUrl: string
    - gameId: string
    - profileId: string
    - branchName: string
    - prUrl: string (or null if no PR created)
    - devOverrideUrl: string (raw.githubusercontent.com URL to the branch profile.json)
    - wikiItemCount: number
    - framesProcessed: number
    - itemsIdentified: number (total from all vision passes before validation)
    - mappedPercent: number (triggers with NCC >= 0.65 / itemsIdentified * 100)
    - retryCount: number (total retry attempts across all items)
    - triggers: Array<{name, triggerId, confidence, nccScore, sourceTimestamp}>
    - newCount, variantCount, skipCount: numbers from dedup

    Report format (markdown):
    ```
    # AI Profile Population Report
    **Game:** {gameId} | **Profile:** {profileId}
    **Source:** {youtubeUrl}
    **Branch:** {branchName}

    ## Coverage
    - Wiki items: {wikiItemCount}
    - Frames processed: {framesProcessed}
    - Items identified: {itemsIdentified}
    - Mapped (NCC ≥ 0.65): {mappedPercent}%
    - Retries: {retryCount}
    - New triggers: {newCount} | Variants: {variantCount} | Skipped (duplicates): {skipCount}

    ## Triggers by Confidence
    ### High (NCC ≥ 0.85)
    {triggers where confidence==='high', listed as: - HH:MM:SS — {name} ({triggerId}) [ncc={nccScore.toFixed(2)}]}
    ### Medium (0.65–0.85)
    {same format}
    ### Needs Review (< 0.65 after retry)
    {same format with ⚠ prefix}

    ## Next Steps
    - Dev override URL: {devOverrideUrl}
    - PR: {prUrl || 'No PR created (0 triggers passed validation)'}
    - To merge: Use popup > Contributor Status > Review Proposals, or accept-proposal Worker op
    ```

    sourceTimestamp format: seconds → HH:MM:SS using Math.floor(t/3600), Math.floor((t%3600)/60), t%60 with zero-padding.

    writeReport(stats, outputPath): calls generateReport, writes to outputPath with fs.writeFileSync. If outputPath not provided, uses './ai-populate-report-{gameId}-{YYYYMMDD-HHmmss}.md' in cwd.

    CLI interface (parse from process.argv manually — no commander needed for 3 flags):
      node index.js <youtube-url> --game <gameId> --profile <profileId> [--interval <seconds>] [--wiki-url <url>] [--model <modelName>] [--output <reportPath>] [--dry-run]
    Required: positional youtube-url, --game, --profile
    --dry-run: runs extraction and vision pass but does NOT commit to GitHub or open PR; still writes report

    Orchestration (following RESEARCH.md architecture, steps 1–10):

    STEP 1 PREFLIGHT:
    - findYtDlp() and findFfmpeg() — catch and print error with install URL then process.exit(1)
    - Check process.env.ANTHROPIC_API_KEY — throw if absent
    - Check process.env.GITHUB_TOKEN — throw if absent (skip in --dry-run mode)
    - Validate youtube-url is non-empty string (guard against shell injection)

    STEP 2 FRAME EXTRACTION:
    - Create tmpdir using os.mkdtemp(path.join(os.tmpdir(), 'ai-populate-'))
    - extractFrames(youtubeUrl, tmpdir, interval) → framePaths

    STEP 3 WIKI GROUNDING:
    - If --wiki-url provided: fetchWikiItems(wikiUrl) else wikiItems = []
    - Log "Wiki items loaded: {count}" (or "No wiki URL provided — skipping wiki grounding")

    STEP 4+5 VISION + CROP + VALIDATE (per frame):
    - For each framePath: read PNG buffer; call identifyItemsInFrame(framePngBuf, wikiItems, {model})
    - For each identified item: rescaleBbox(bbox, 1920, 1080); validateCrop(framePngBuf, bbox.x, bbox.y, bbox.w, bbox.h, 8)
    - On NCC < 0.65: retry once with validateCrop using padding=2 (tighter crop)
    - Collect results as candidateTriggers array with { name, cropBuf, cropW, cropH, nccScore, confidence, sourceTimestamp (frame index * interval), frameW:1920, frameH:1080 }
    - Track retryCount and itemsIdentified for report stats

    STEP 6 DEDUP:
    - Build branchName = makeBranchName(gameId, profileId)
    - In non-dry-run: gh = githubClient(process.env.GITHUB_TOKEN); result = await createOrGetBranch(gh, branchName)
    - If existed: load existing profile via loadBranchProfile; build pngsByTriggerId map from existing references
    - deduplicateTriggers(candidateTriggers, existingProfile, pngsByTriggerId) → { triggers, newCount, variantCount, skipCount }

    GUARD: If triggers.length === 0 after dedup (0 passed validation): print summary, write report, exit 0 with message "No triggers passed validation. Profile branch not created." Do NOT open PR.

    STEP 7 PROFILE BRANCH BUILD (skip in --dry-run):
    - Build profile.json from triggers array (schema per CLAUDE.md: { triggers: [{id, payloads, references, rotation, rotates}] })
    - commitProfile(gh, gameId, profileId, branchName, profileJson, referencePngs map)

    STEP 8 VALIDATION PASS:
    - For each trigger: re-read the source frame PNG (from tmpdir by timestamp index); validateCrop with the trigger's bbox; log "PASS" or "FAIL" per trigger

    STEP 9 PR CREATION (skip in --dry-run):
    - openPR(gh, branchName, gameId, profileId, triggers.length) → prUrl

    STEP 10 SUMMARY:
    - devOverrideUrl = "https://raw.githubusercontent.com/frothydv/streamGenieProfiles/{branchName}/games/{gameId}/profiles/{profileId}/profile.json"
    - Build stats object; writeReport(stats, outputPath); print report to console

    CLEANUP: fs.rmSync(tmpdir, { recursive: true, force: true }) in a finally block.

|----------|-------------|
| CLI args → child processes | youtube-url passed as spawn array arg to yt-dlp — never interpolated |
| vision API response → filesystem | Crops from AI bbox are clamped and validated before writing to disk |
| GitHub token → API | Only used for this session; never logged |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-11 | Tampering | youtube-url arg → yt-dlp spawn | mitigate | spawnSync called with args array; URL validated as non-empty string before spawn |
| T-05-12 | Information Disclosure | GITHUB_TOKEN / ANTHROPIC_API_KEY in report | mitigate | generateReport never receives or outputs token values; preflight checks existence only |
| T-05-13 | Tampering | zero-trigger guard bypassed | mitigate | Explicit length === 0 check before commitProfile; guard cannot be bypassed by dedup logic |
| T-05-14 | Denial of Service | tmpdir not cleaned on crash | mitigate | fs.rmSync in finally block ensures cleanup even on exception |
</threat_model>
