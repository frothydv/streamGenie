<interfaces>
From extension/matcher-core.js (UMD module, works in Node via module.exports):
- createMatcher() → matcher instance
- matcher.buildRefNCC(pixels: Uint8Array, w: number, h: number) → refNCC object
- matcher.buildSAT(grayBuffer: Float32Array, w: number, h: number) → { sat, sat2 }
- matcher.nccScoreAt(sceneGray: Float32Array, sceneW: number, sat, sat2, bboxX: number, bboxY: number, refNCC, refW: number, refH: number) → number (0–1)
- matcher.dHashFromPixels(pixels: Uint8Array, w: number, h: number) → Uint8Array(64)

Note: To fill a Float32Array with grayscale values from RGBA pixels, use:
  for (let i = 0; i < w * h; i++) { gray[i] = (r*0.299 + g*0.587 + b*0.114) / 255; }
  where r = pixels[i*4], g = pixels[i*4+1], b = pixels[i*4+2].

From pngjs (already in node_modules):

    - validateCrop(framePngBuf: Buffer, bboxX: number, bboxY: number, bboxW: number, bboxH: number, padding=4): crops the frame at bbox+padding (clamped to frame bounds), builds refNCC from the crop, then runs nccScoreAt with the FULL FRAME as scene at position (bboxX-padding, bboxY-padding); returns { score: number, cropBuf: Buffer, cropW, cropH }
    - confidenceTier(score): returns 'high' (score >= 0.85), 'medium' (0.65 <= score < 0.85), 'needs-review' (score < 0.65)
    - Bbox coordinates are clamped: x2 = Math.min(frameW, bboxX + bboxW + padding); x1 = Math.max(0, bboxX - padding); same for y
    - CRITICAL: scene passed to nccScoreAt is the full frame's Float32Array, not the crop (per RESEARCH.md Pitfall 3 and critical_research_findings item 5)
    - Returns cropBuf as a valid PNG buffer (PNG.sync.write output)
  </behavior>
  <action>
    Create scripts/ai-populate/lib/validate.js. Export validateCrop and confidenceTier (module.exports = { validateCrop, confidenceTier }).

    validateCrop implementation:
    1. PNG.sync.read(framePngBuf) → frameW, frameH, frameData
    2. Build full-frame Float32Array: for each pixel i, gray[i] = (r*0.299 + g*0.587 + b*0.114) / 255
    3. Build SAT for full frame: matcher.buildSAT(frameGray, frameW, frameH) → { sat, sat2 }
    4. Clamp crop coords: x1 = max(0, bboxX-padding), y1 = max(0, bboxY-padding), x2 = min(frameW, bboxX+bboxW+padding), y2 = min(frameH, bboxY+bboxH+padding); cropW = x2-x1, cropH = y2-y1
    5. Extract crop RGBA pixels from frameData into a new Uint8Array
    6. Build refNCC for the crop: matcher.buildRefNCC(cropPixels, cropW, cropH)
    7. Score: matcher.nccScoreAt(frameGray, frameW, sat, sat2, x1, y1, refNCC, cropW, cropH)
    8. Write crop back to PNG buffer for storage
    9. Return { score, cropBuf, cropW, cropH }
